## Descrição

A Contourline quer expandir internacionalmente com base em dados, não em intuição. O CEO Egio acredita que "estamos passando pela maior mudança da nossa era" e quer usar IA como diferencial competitivo. A certificação Anvisa é o passaporte: agências regulatórias de outros países a reconhecem como indicador de qualidade. Mas a empresa não tem processo estruturado de inteligência de mercado para decidir onde entrar.

Esta story implementa um sistema de inteligência de mercado assistida por IA: pesquisa estruturada de mercados potenciais, scoring automatizado baseado no framework definido na story 10.1, e relatórios executivos gerados por LLM com base em dados de fontes abertas (regulação local, tamanho de mercado, concorrentes, barreiras de entrada).

O output é acionável: ao final de cada ciclo de pesquisa (trimestral), a Contourline tem um relatório ranqueado dos mercados mais atrativos com recomendação fundamentada de onde focar esforços.

Meta do epic: 3+ novos mercados avaliados por trimestre.

Stack: N8N (orquestração de pesquisa) + Supabase (base de dados de mercados) + OpenAI/Claude API (síntese e análise) + Metabase (dashboard de mercados) + fontes abertas (OECD, Banco Mundial, regulatórias locais, Anvisa internacional).

## Dimensões de Pesquisa por Mercado

| Dimensão | Dados Coletados | Fonte |
|----------|----------------|-------|
| Tamanho de Mercado | TAM de equipamentos médicos estéticos | OECD, IBISWorld, relatórios setoriais |
| Regulação | Agência regulatória, processo de registro, equivalência Anvisa | Site da agência regulatória local |
| Concorrência | Quem opera, quais produtos, faixa de preço | Pesquisa web, LinkedIn, feiras |
| Parcerias | Distribuidores locais potenciais, associações médicas | LinkedIn, associações, câmaras de comércio |
| Ambiente Econômico | PIB per capita, crescimento, câmbio, risco país | Banco Mundial, FMI |
| Infraestrutura | Logística, suporte local possível | Análise operacional |
| Cultura | Distância cultural, idioma, comportamento do ICP local | Pesquisa qualitativa |

## Tasks Técnicas

**T10.5.1 — Formalizar e documentar o framework de pesquisa**
- Baseado no framework da story 10.1 (7 dimensões + scoring)
- Criar template de pesquisa: o que pesquisar em cada dimensão, fontes recomendadas, como pontuar
- Template executável por Filipe sem suporte de TI

**T10.5.2 — Criar schema de inteligência de mercado no Supabase**
- Tabela `internacional_mercados` com: pais, regiao, data_pesquisa, pesquisado_por, status_pesquisa (identificado/em_pesquisa/avaliado/descartado/ativo), score_total, pontuacoes_por_dimensao (JSON), recomendacao, notas
- Tabela `internacional_pesquisa_dados` com: mercado_id, dimensao, dado_coletado, fonte_url, data_coleta, confianca (alta/media/baixa)
- Tabela `internacional_relatorios` com: mercado_id, data, relatorio_texto, gerado_por_ia, revisado_por, aprovado

**T10.5.3 — Implementar workflow de pesquisa assistida por N8N**
- Interface de início de pesquisa: Filipe preenche formulário com país-alvo
- N8N dispara sequência de coleta:
  - Busca web via EXA (MCP) sobre regulação local de dispositivos médicos
  - Busca sobre tamanho de mercado (relatórios públicos)
  - Busca sobre principais concorrentes no país
  - Busca sobre distribuidores de equipamentos médicos no país
- Dados coletados → INSERT no Supabase

**T10.5.4 — Implementar geração de relatório por IA**
- Após coleta de dados: N8N monta prompt contextualizado com todos os dados coletados
- Prompt para LLM (Claude/GPT): "Analise os dados coletados sobre o mercado {país} para uma empresa brasileira de equipamentos médicos estéticos, certificada pela Anvisa, faturamento R$115M. Avalie viabilidade de expansão considerando: regulação, concorrência, parceiros potenciais, barreiras. Gere relatório com: sumário executivo, pontuação por dimensão, recomendação (entrar/aguardar/descartar) e próximos passos."
- Relatório gerado → INSERT em `internacional_relatorios`
- E-mail para Filipe + link para revisão humana antes de compartilhar

**T10.5.5 — Criar processo de revisão humana do relatório**
- Filipe Ferreira revisa o relatório gerado por IA antes de distribuir
- Interface simples: link para aprovar ou solicitar ajustes
- Após aprovação: relatório marcado como revisado; disponível no dashboard e para envio

**T10.5.6 — Criar scoring automático por dimensão**
- Após coleta e análise: sistema calcula score por dimensão (1-5) baseado em heurísticas configuráveis
- Score total calculado (soma ponderada)
- Classificação: ALTA PRIORIDADE (>= 35/50), MÉDIA PRIORIDADE (25-34), BAIXA PRIORIDADE (<25)
- INSERT de pontuações no Supabase

**T10.5.7 — Criar dashboard de inteligência de mercado no Metabase**
- Mapa de mercados por status (identificado/em pesquisa/avaliado/descartado/ativo)
- Ranking de mercados avaliados por score total
- Comparativo de dimensões entre mercados (radar chart por mercado)
- Timeline de pesquisas realizadas (meta: 3+ por trimestre)
- Filtros por região, score, status

**T10.5.8 — Configurar ciclo trimestral de pesquisa**
- N8N cron: primeiro dia de cada trimestre → lembrete para Filipe de iniciar novo ciclo de pesquisa
- Meta: 3 novos mercados pesquisados por trimestre
- Relatório consolidado trimestral: todos os mercados avaliados, ranking atualizado, recomendações
