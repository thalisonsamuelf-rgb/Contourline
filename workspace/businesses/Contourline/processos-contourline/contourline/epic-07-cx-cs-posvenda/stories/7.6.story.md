## Descrição

A receita de expansão (upsell e cross-sell) é hoje não estruturada na Contourline. O CRM (RD Station) é usado principalmente para o funil pré-venda, mas após a compra o contato do cliente não gera oportunidades formais de expansão. A meta é +15% de receita de expansão — o que representa R$17-18M adicionais no faturamento anual.

Esta story implementa um funil dedicado de upsell e expansão no RD Station, com gatilhos automáticos baseados em comportamento do cliente: conclusão do treinamento LMS, NPS alto (promotor), tempo de uso do equipamento e sinais de crescimento do consultório. O fluxo conecta CX com Comercial para fechar oportunidades de expansão.

O upsell na Contourline pode ser: novo equipamento, acessórios e consumíveis, extensão de suporte, novos protocolos (cursos avançados no LMS), ou outros equipamentos da linha para o mesmo consultório.

Stack: RD Station (funil de upsell) + N8N (automação de gatilhos) + Supabase (dados de comportamento).

## Tipos de Gatilho de Upsell

| Gatilho | Critério | Tipo de Upsell Sugerido |
|---------|----------|------------------------|
| NPS Promotor | Score >= 9 na pesquisa D+90 | Indicação + novo equipamento |
| LMS Concluído | 100% trilha + certificado emitido | Curso avançado + acessórios |
| Aniversário de 1 ano | 365 dias desde primeira compra | Renovação + novo produto |
| Score churn BAIXO por 6 meses | Score < 20 consistentemente | Cross-sell de linha complementar |
| Sem chamados de suporte (3 meses) | Zero chamados abertos | Upgrade de equipamento |

## Tasks Técnicas

**T7.6.1 — Configurar funil de upsell no RD Station**
- Criar pipeline dedicado "Expansão e Upsell" separado do funil comercial principal
- Etapas: Potencial Identificado → Em Qualificação → Abordagem Ativa → Proposta Enviada → Ganho / Perdido
- Campos customizados: tipo de upsell, produto alvo, receita estimada, gatilho de origem

**T7.6.2 — Criar catálogo de oportunidades de expansão no Supabase**
- Tabela `cx_upsell_catalog` com: produto_id (referência Totvs), categoria (equipamento/acessório/curso/suporte), descricao, preco_referencia, prerequisitos (ex: equipamento base necessário)
- Relação com produtos Totvs para consistência de dados

**T7.6.3 — Implementar engine de detecção de gatilhos no N8N**
- Workflow cron diário que avalia todos os clientes ativos
- Verificação de cada tipo de gatilho (lista acima)
- Se gatilho detectado: verifica se oportunidade já existe para este cliente/produto (evitar duplicatas)
- Se novo: cria deal no RD Station via API + INSERT em `cx_upsell_opportunities`

**T7.6.4 — Criar tabela de oportunidades de upsell no Supabase**
- Tabela `cx_upsell_opportunities` com: cliente_id, gatilho_tipo, produto_recomendado, deal_id_crm, status, data_identificacao, data_fechamento, valor_estimado, consultor_responsavel
- Sync periódico com status atual do deal no CRM

**T7.6.5 — Configurar notificacoes para Mariana Normando (Analista de Experiencia e Relacionamento) e time CX**
- E-mail imediato quando nova oportunidade de upsell criada para sua carteira
- Mensagem inclui: nome cliente, equipamento/produto recomendado, gatilho que acionou, script sugerido de abordagem
- Lembrete apos 5 dias sem acao na oportunidade

**T7.6.6 — Criar scripts de abordagem por tipo de upsell**
- Template de e-mail para cada tipo de upsell (NPS promotor, aniversário, conclusão LMS)
- Template de script para ligação ou WhatsApp
- Validar com time Comercial e CX

**T7.6.7 — Criar dashboard de expansão no Metabase**
- Pipeline de upsell: quantidade por etapa (funil visual)
- Receita estimada total do pipeline
- Taxa de conversão de upsell (% oportunidades → ganho)
- Receita de expansão realizada no mês (via Totvs)
- Top 3 produtos de upsell mais convertidos
- Ranking de Consultores por receita de expansão

**T7.6.8 — Integrar com dados de compras do Totvs**
- Via N8N: consultar histórico de compras do cliente no Totvs (dados carregados no Supabase via ETL da story 9.x)
- Usar histórico para identificar produtos complementares (ex: cliente tem equipamento A → recomendar acessório B)
