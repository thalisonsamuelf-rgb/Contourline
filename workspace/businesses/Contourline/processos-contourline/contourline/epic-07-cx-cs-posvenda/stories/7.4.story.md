## Descrição

A Contourline não possui hoje um sistema estruturado de coleta de feedback dos clientes. Não há NPS medido, não há pesquisas de satisfação automatizadas, e não há dados de retenção baseados em percepção do cliente. Isso cria um ponto cego crítico: churns ocorrem sem que o time de CX tenha sinal antecipado.

Esta story implementa um sistema automatizado de NPS e pesquisas de satisfação, integrado ao fluxo de pós-venda. As pesquisas são disparadas em momentos estratégicos da jornada do cliente (definidos na story 7.1), os resultados são consolidados no Supabase e visualizados no Metabase, com alertas para detratores.

O design das pesquisas deve considerar o ICP: profissional ocupado de 35-55 anos que não tem tempo para formulários longos. Máximo 3 perguntas por pesquisa, preferencialmente respondível em 1 clique via e-mail ou WhatsApp.

Stack: N8N (disparo de pesquisas) + Typeform ou Google Forms (coleta) + Supabase (consolidação) + Metabase (dashboard NPS).

---

## Acceptance Criteria

- [ ] NPS coletado automaticamente em 3 momentos: D+30 (pós-onboarding), D+90 (uso regular), D+180 (expansão)
- [ ] Pesquisa de satisfação pós-suporte disparada automaticamente após cada chamado resolvido
- [ ] Formulários com no máximo 3 perguntas, mobile-first, respondíveis em menos de 2 minutos
- [ ] Resposta ao NPS registrada automaticamente no Supabase em até 5 minutos após envio
- [ ] Dashboard Metabase com: NPS geral, NPS por produto, evolução mensal, distribuição promotores/neutros/detratores
- [ ] Alerta automático para time CX quando NPS < 6 (detrator) em até 1h após resposta
- [ ] Alerta para gestor quando NPS médio cair abaixo de 40 em qualquer período de 30 dias
- [ ] Taxa de resposta > 30% após 90 dias de uso (benchmark para pesquisas B2B)

## Tasks Técnicas

**T7.4.1 — Definir modelo de pesquisas e perguntas**
- NPS padrão: "Em uma escala de 0-10, quanto você recomendaria a Contourline a um colega?"
- Pergunta aberta opcional: "O que poderíamos melhorar?"
- Pesquisa pós-suporte: "Seu problema foi resolvido? (Sim/Não)" + "Como foi o atendimento? (1-5)"
- Validar perguntas com time CX

**T7.4.2 — Selecionar e configurar ferramenta de pesquisa**
- Avaliar: Typeform (integração API fácil), Google Forms (gratuito, menos design), Surveymonkey
- Configurar formulários com branding Contourline
- Testar envio por e-mail e WhatsApp

**T7.4.3 — Criar flows N8N para disparo de pesquisas**
- Flow 1: NPS D+30 — trigger: 30 dias após deal ganho no CRM
- Flow 2: NPS D+90 — trigger: 90 dias após deal ganho
- Flow 3: NPS D+180 — trigger: 180 dias após deal ganho
- Flow 4: Pesquisa pós-suporte — trigger: chamado marcado como resolvido
- Cada flow: busca dados do cliente, personaliza mensagem, envia por e-mail + WhatsApp

**T7.4.4 — Criar schema no Supabase**
- Tabela `cx_nps_responses` com: cliente_id, tipo_pesquisa (NPS/pos-suporte), score, comentario, data_resposta, canal
- Tabela `cx_survey_dispatches` com: cliente_id, tipo, data_envio, status (enviado/respondido/expirado)
- Views analíticas: NPS_score_geral, NPS_por_produto, tendencia_mensal

**T7.4.5 — Criar webhook de recepção de respostas**
- Endpoint N8N recebe resposta do Typeform/Google Forms via webhook
- Parser dos dados + INSERT no Supabase
- Regra de alerta para detratores (NPS < 6)

**T7.4.6 — Configurar alertas automáticos**
- Alerta detrator: resposta NPS < 6 → notificação imediata para responsável CX do cliente (e-mail + WhatsApp)
- Alerta tendencia: NPS medio < 40 nos ultimos 30 dias → e-mail para Andreia Almeida (Diretora de Consultoria e Treinamentos) e Xavier Ramos (Gerente Regional CS/CX)
- Template de resposta padrão para detratores (CX aciona o cliente em 24h)

**T7.4.7 — Criar dashboard NPS no Metabase**
- Painel principal: NPS Score atual (grande), evolução 6 meses (gráfico linha), distribuição (donut chart)
- Painel detalhado: por produto, por Consultora responsável, por tempo de cliente
- Lista de detratores recentes com dados de contato
- Lista de promotores (candidatos a case/indicação)

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Taxa de resposta abaixo de 15% | Média | Médio | Testar horário de envio; usar WhatsApp como canal principal |
| NPS inicial muito baixo expondo problemas não mapeados | Média | Médio | Ter plano de ação para cada faixa de NPS antes de lançar |
| Fadiga de pesquisas (cliente recebendo muitas) | Baixa | Médio | Limitar a 1 pesquisa por mês por cliente |
| Integração Typeform/Forms com N8N instável | Baixa | Baixo | Usar webhook nativo como backup |
