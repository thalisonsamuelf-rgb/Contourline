## Contexto

A Contourline comercializa equipamentos medicos de alto valor com certificacao Anvisa obrigatoria. A rastreabilidade por numero de serie e lote e um requisito regulatorio inegociavel — nao uma feature opcional. Atualmente, o controle de estoque e feito parcialmente no Totvs e parcialmente em planilhas, com gaps de rastreabilidade que representam risco de perda de certificacao.

Esta story implementa o modulo de gestao de estoque integrado ao Totvs, com rastreabilidade completa por numero de serie, dashboard de posicao de estoque em tempo real e alertas automaticos de niveis criticos. O sistema deve operar complementando o Totvs, nunca substituindo-o.

Stack: Supabase (PostgreSQL) como data warehouse, N8N para integracao com Totvs, Metabase para visualizacao.

## Objetivo

Implementar sistema de gestao de estoque com rastreabilidade Anvisa por numero de serie, integrado ao Totvs via N8N, com visibilidade em tempo real via Metabase e alertas automaticos — eliminando os gaps regulatorios atuais e reduzindo ruptura de estoque para menos de 2%.

## Acceptance Criteria

- [ ] AC1: Cadastro de produtos com numero de serie, lote, data de fabricacao e certificado Anvisa no Supabase
- [ ] AC2: Registro de entrada de estoque (recebimento) com rastreabilidade completa por item
- [ ] AC3: Registro de saida de estoque (expedicao) vinculado a pedido de venda com numero de serie
- [ ] AC4: Integracao bidirecional com Totvs via N8N: sincronizacao de movimentacoes de estoque em batch diario
- [ ] AC5: Dashboard de posicao de estoque em tempo real no Metabase (quantidade, localizacao, status)
- [ ] AC6: Alerta automatico (email + WhatsApp) quando estoque atinge nivel minimo configurado por SKU
- [ ] AC7: Rastreabilidade completa: dado um numero de serie, exibir historico completo (entrada, localizacao, saida, destinatario)
- [ ] AC8: Modulo de inventario: suporte a contagem ciclica com reconciliacao automatica vs Totvs
- [ ] AC9: Acuracidade de estoque validada em >= 98% apos go-live (teste com amostra de 100 itens)
- [ ] AC10: Historico de movimentacoes auditavel por 5 anos (requisito Anvisa)

## Tasks Tecnicas

**T4.2.1** — Design do schema de dados no Supabase: tabelas `products`, `stock_items`, `stock_movements`, `locations`, `anvisa_certificates`

**T4.2.2** — Implementar RLS policies no Supabase: separar acesso por perfil (operador de estoque, gestor, diretor)

**T4.2.3** — Desenvolver workflow N8N `totvs-stock-sync`: extrair movimentacoes do Totvs e sincronizar com Supabase em batch diario (00:30)

**T4.2.4** — Implementar API de entrada de estoque: endpoint POST `/stock/receive` com validacao de certificado Anvisa

**T4.2.5** — Implementar API de saida de estoque: endpoint POST `/stock/dispatch` vinculando numero de serie a pedido de venda

**T4.2.6** — Desenvolver workflow N8N de alerta de estoque minimo: checar niveis a cada 4h e disparar notificacoes

**T4.2.7** — Criar dashboard Metabase "Posicao de Estoque": cards de quantidade total, itens abaixo do minimo, movimentacoes do dia

**T4.2.8** — Implementar funcao de rastreabilidade: busca por numero de serie retorna historico completo com timestamps

**T4.2.9** — Desenvolver modulo de inventario ciclico: interface para contagem e workflow N8N de reconciliacao automatica

**T4.2.10** — Configurar retencao de dados: policy de 5 anos para movimentacoes de estoque (compliance Anvisa)

**T4.2.11** — PoC de integracao Totvs: validar conectividade e formato de dados antes do desenvolvimento completo

**T4.2.12** — Testes de carga: simular 1.000 movimentacoes/dia e validar performance (<2s por operacao)

**T4.2.13** — Documentacao tecnica: runbook de operacao e procedimento de contingencia para falha de sincronizacao

## Riscos e Mitigacao

| Risco | Probabilidade | Impacto | Mitigacao |
|-------|--------------|---------|-----------|
| API do Totvs indisponivel ou sem documentacao adequada | Media | Alto | Realizar PoC (T4.2.11) antes do desenvolvimento; preparar fallback via exportacao manual CSV |
| Divergencia entre dados do Totvs e realidade do estoque fisico | Alta | Alto | Periodo de dupla operacao: 30 dias com sistema novo e Totvs em paralelo |
| Numero de serie nao cadastrado no Totvs para itens antigos | Media | Medio | Processo de migracao/cadastro retroativo antes do go-live |
| Falha na sincronizacao N8N nao detectada | Baixa | Alto | Monitoramento com alertas de falha e dashboard de status de sync |
