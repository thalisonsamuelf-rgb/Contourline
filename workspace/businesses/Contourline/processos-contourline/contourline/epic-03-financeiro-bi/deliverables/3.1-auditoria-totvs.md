## Sumario Executivo

Esta auditoria mapeia os dados financeiros disponiveis no Totvs da Contourline, avaliando capacidades de API, entidades criticas por subarea, limitacoes conhecidas e planos alternativos para extracao de dados. O objetivo e habilitar o Epic 03 (camada de IA sobre o Totvs) sem substituir o ERP existente.

O risco principal do Epic 03 -- "API do Totvs limitada ou inexistente" -- e tratado neste documento com estrategia primaria (API REST) e plano B (exportacao periodica).

**Descoberta relevante:** Ja existe um banco de dados auxiliar (`contourline-db`) com entidades de Clientes, Equipamentos, Vendas, Chamados e Academy, integrado com RD Station CRM. Este banco e uma fonte complementar ao Totvs para dados comerciais.

## 2. APIs Disponiveis do Totvs

### 2.1 Totvs Protheus -- API REST (cenario mais provavel)

O Protheus possui API REST nativa a partir das versoes mais recentes (12.1.27+). A plataforma "TOTVS API" segue o padrao TBC (TOTVS Business Connector).

#### Endpoints Financeiros Esperados

| Endpoint / API | Descricao | Disponibilidade |
|----------------|-----------|-----------------|
| `/api/fin/v1/accountsPayable` | Contas a Pagar -- titulos, vencimentos, fornecedores | Protheus 12.1.27+ |
| `/api/fin/v1/accountsReceivable` | Contas a Receber -- titulos, clientes, aging | Protheus 12.1.27+ |
| `/api/fin/v1/bankTransactions` | Movimentacoes bancarias | Protheus 12.1.27+ |
| `/api/fin/v1/cashFlow` | Fluxo de caixa | Pode exigir customizacao |
| `/api/ctb/v1/accountingEntries` | Lancamentos contabeis | Protheus 12.1.27+ |
| `/api/ctb/v1/chartOfAccounts` | Plano de contas | Protheus 12.1.27+ |
| `/api/fis/v1/taxDocuments` | Documentos fiscais (NF-e) | Protheus 12.1.27+ |
| `/api/com/v1/purchaseOrders` | Ordens de compra | Protheus 12.1.27+ |
| `/api/com/v1/suppliers` | Cadastro de fornecedores | Protheus 12.1.27+ |
| `/api/rh/v1/payroll` | Folha de pagamento | Protheus com modulo GPE |
| `/api/rh/v1/employees` | Cadastro de funcionarios | Protheus com modulo GPE |
| `/api/fat/v1/invoices` | Faturamento | Protheus 12.1.27+ |
| `/api/est/v1/stockPositions` | Posicoes de estoque | Protheus 12.1.27+ |

#### Autenticacao

| Metodo | Descricao |
|--------|-----------|
| **Basic Auth** | Usuario e senha do Protheus (mais comum em versoes 12.x) |
| **OAuth 2.0** | Disponivel em Protheus 12.1.33+ com TOTVS Identity |
| **API Key** | Algumas customizacoes usam token fixo |

#### Rate Limits e Paginacao

| Parametro | Valor Tipico |
|-----------|-------------|
| Rate limit | 100-500 req/min (configuravel no AppServer) |
| Paginacao | `page` + `pageSize` (max 100-200 registros por pagina) |
| Timeout | 30-120 segundos por requisicao |
| Formato resposta | JSON (padrao TOTVS API) |
| Bulk export | Nao nativo -- requer queries paginadas |

### 2.2 Totvs RM -- API REST (cenario alternativo)

| Endpoint / API | Descricao |
|----------------|-----------|
| `/api/framework/v1/` | APIs do framework RM |
| `/api/financeiro/v1/contasPagar` | Contas a pagar |
| `/api/financeiro/v1/contasReceber` | Contas a receber |
| `/api/contabil/v1/lancamentos` | Lancamentos contabeis |
| DataServer (SOAP) | Camada legada com acesso direto aos DataServers |

### 2.3 TOTVS RAC (Plataforma Unificada)

A partir de 2024, a Totvs vem unificando APIs sob a plataforma **TOTVS RAC** (TOTVS App Center), com APIs padronizadas e marketplace de integracao:

| Recurso | Status |
|---------|--------|
| TOTVS Developer | Portal de documentacao de APIs |
| TOTVS Connect | Middleware de integracao (iPaaS) |
| Webhooks | Disponivel para alguns eventos |
| APIs padronizadas (TOTVS API Guide) | Em expansao progressiva |

## 5. Limitacoes Conhecidas da API Totvs

### 5.1 Limitacoes Estruturais

| Limitacao | Impacto | Severidade |
|-----------|---------|-----------|
| **API REST pode nao estar habilitada** | Sem acesso programatico. Impede toda a estrategia API-first. | Critica |
| **Versao antiga do Protheus (< 12.1.27)** | APIs REST nao disponiveis nativamente. Apenas SOAP/DataServer. | Critica |
| **Sem TOTVS Identity / OAuth** | Autenticacao limitada a Basic Auth, menor seguranca | Alta |
| **Rate limits restritivos** | Exportacao de grandes volumes lenta (DRE com milhares de lancamentos) | Alta |
| **Sem webhooks nativos** | Sem notificacao de eventos em tempo real. Polling necessario. | Media |
| **Customizacoes locais alteram schema** | Campos customizados podem nao estar expostos na API padrao | Alta |
| **API somente leitura para alguns modulos** | Escrita no Totvs via API pode nao estar disponivel | Media (Epic 03 e leitura) |
| **Documentacao variavel** | APIs customizadas podem nao ter documentacao | Media |

### 5.2 Limitacoes por Modulo

| Modulo | Limitacao Esperada | Workaround |
|--------|-------------------|-----------|
| Fluxo de Caixa projetado | Raramente exposto via API | Calcular a partir de AP + AR + saldos |
| DRE / Balanco | Relatorios consolidados nao tem API direta | Gerar a partir de lancamentos contabeis |
| Folha detalhada | Dados sensiveis, pode ter restricao de API | Exportacao com controle de acesso |
| Historico de cobranca | Registros de contato raramente na API | Campo customizado ou exportacao |
| Budget / Orcamento | Nem sempre digitalizado no Totvs | Importar de planilha Excel |
| Cotacoes de compras | Pode estar fora do Totvs | Novo sistema (Story 3.5) |

### 5.3 Impacto por Story

| Story | Impacto se API Limitada | Severidade |
|-------|------------------------|-----------|
| 3.2 Dashboard | Alto -- sem dados real-time, dashboard fica com dados defasados | Critica |
| 3.3 Relatorios | Medio -- relatorios podem ser gerados com dados exportados (batch) | Alta |
| 3.4 Cobranca | Alto -- regua de cobranca precisa de dados de aging atualizados | Alta |
| 3.5 Compras | Medio -- compras pode ser sistema novo, independente do Totvs | Media |
| 3.6 Folha | Medio -- dados de folha podem vir de exportacao mensal | Media |
| 3.7 BI Central | Alto -- pipeline ETL precisa de acesso programatico | Alta |

## 6. Plano B -- Estrategias para Dados sem API

### 6.1 Exportacao Periodica (Batch)

| Estrategia | Descricao | Frequencia | Automacao |
|-----------|-----------|-----------|----------|
| **Export CSV/Excel do Protheus** | Relatorios padrao exportados como arquivo | Diario ou semanal | Agendamento via Protheus (Schedule) |
| **Export via SQL direto** | Query no banco do Totvs (SQL Server/Oracle) | Diario | Script agendado + controle de acesso |
| **CNAB / Arquivo bancario** | Arquivos de retorno bancario | Diario | Ja existente para cobranca |
| **SPED / Obrigacoes acessorias** | Arquivos XML/TXT gerados para o fisco | Mensal | Reutilizar para relatorios internos |

### 6.2 Integracao por Banco de Dados (DB Link)

| Metodo | Descricao | Risco |
|--------|-----------|-------|
| **Read-only replica** | Replica do banco Totvs apenas para leitura | Baixo risco, depende de infra |
| **View materializada** | Views no banco Totvs que consolidam dados | Medio -- pode impactar performance |
| **ETL direto no banco** | Ferramenta de ETL acessa banco Totvs | Medio -- bypass de regras de negocio |

**RECOMENDACAO:** DB Link com read-only replica e a estrategia de Plano B mais robusta. Permite queries SQL completas sem impacto no Totvs de producao.

### 6.3 Webhooks Customizados

| Metodo | Descricao | Complexidade |
|--------|-----------|-------------|
| **Trigger no banco Totvs** | Trigger SQL dispara notificacao em evento (pagamento, lancamento) | Alta -- requer acesso DBA |
| **Middleware Totvs (TOTVS Connect)** | Se disponivel, configura webhooks por evento | Media -- depende do contrato |
| **Polling programado** | Consulta periodica de novos registros (delta query) | Baixa -- funciona com qualquer API |

### 6.4 RPA / Scraping (Ultimo Recurso)

| Metodo | Descricao | Quando Usar |
|--------|-----------|------------|
| **RPA (Robotic Process Automation)** | Bot navega na interface do Totvs e extrai dados | Somente se API e DB Link nao funcionam |
| **Scraping de relatorios** | Parse automatico de PDFs/planilhas gerados pelo Totvs | Somente para relatorios sem alternativa |

**ALERTA:** RPA e scraping sao frageis (quebram com atualizacoes de UI) e devem ser evitados. Sao o ultimo recurso, nao a primeira opcao.

### 6.5 Decisao de Estrategia por Cenario

| Cenario | Estrategia Primaria | Plano B | Plano C |
|---------|--------------------|---------|---------
| API REST disponivel e funcional | API REST com polling | DB Link read-only | Export batch |
| API REST parcial (alguns modulos) | API para modulos disponiveis | DB Link para o restante | Export batch |
| Sem API REST (versao antiga) | DB Link read-only | Export batch automatizado | RPA (ultimo recurso) |
| Cloud Totvs (SaaS) | API REST (unica opcao) | Export programado | TOTVS Connect |

## 7. Requisitos de Seguranca para Dados Financeiros

### 7.1 Classificacao de Dados

| Categoria | Exemplos | Nivel de Sensibilidade |
|-----------|---------|----------------------|
| Dados financeiros operacionais | AP, AR, fluxo de caixa, DRE | **Confidencial** |
| Dados de folha e pessoais | Salarios, CPF, dados bancarios de funcionarios | **Altamente Confidencial** (LGPD) |
| Dados fiscais | Notas fiscais, SPED, obrigacoes | **Confidencial** |
| Dados de fornecedores | Cadastro, condicoes comerciais | **Interno** |
| Dados de clientes (financeiro) | Inadimplencia, historico de pagamento | **Confidencial** |

### 7.2 Requisitos Minimos de Seguranca

| Requisito | Descricao | Obrigatorio |
|-----------|-----------|------------|
| **Criptografia em transito** | TLS 1.2+ para todas as comunicacoes API | Sim |
| **Criptografia em repouso** | Dados financeiros no BI central criptografados | Sim |
| **Controle de acesso por perfil** | CFO ve tudo; outros financeiros veem seu escopo | Sim |
| **Audit trail** | Log de quem acessou quais dados e quando | Sim |
| **Principio do menor privilegio** | Acesso API somente leitura; sem permissao de escrita no Totvs | Sim |
| **Segregacao de ambientes** | Dados de producao nunca em ambientes de teste/dev | Sim |
| **Backup e retencao** | Dados do BI central com backup diario e retencao 5 anos | Sim |
| **Mascaramento de dados** | CPF, dados bancarios mascarados em ambientes nao-producao | Sim |
| **LGPD compliance** | Base legal para tratamento de dados pessoais (folha) | Sim |
| **IP Allowlist** | API Totvs acessivel somente de IPs autorizados | Recomendado |
