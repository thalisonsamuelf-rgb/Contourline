## Descrição

A Contourline opera com múltiplos sistemas de dados que não se comunicam: Totvs (ERP/Financeiro), RD Station (CRM/Comercial), ferramentas de Marketing, e outros sistemas setoriais. O Epic 09 (TI) tem como objetivo criar um hub de integração central — e esta story implementa o pilar financeiro desse hub: a integração do Totvs com o BI central da empresa.

O BI central é o lugar onde dados de todos os setores se encontram para análises cruzadas: qual o custo de aquisição de cliente (dado do CRM + dado financeiro), qual a margem por produto (dado do Totvs + dado comercial), como o forecast de vendas se relaciona com o fluxo de caixa projetado.

Esta story é a mais estratégica e a mais longa do Epic 03, e deve ser coordenada com a Story 9.2 (Arquitetura de dados do Epic 09) para garantir que o design está alinhado com o BI central planejado.

---

## Acceptance Criteria

- [ ] Pipeline de integração Totvs → BI central operacional (batch ou streaming, conforme definido com @architect)
- [ ] Entidades financeiras críticas disponíveis no BI central: DRE, Balanço, Fluxo de Caixa, Contas a Pagar/Receber, Folha (resumo), Compras
- [ ] Dados financeiros reconciliados com o Totvs: o que está no BI central bate com o que está no Totvs (tolerância de divergência < 0.1%)
- [ ] Latência de dados documentada e aceitável: definir com CFO se dados near-real-time (1h) ou diários são suficientes
- [ ] Análises cruzadas possíveis: CAC (custo de aquisição de cliente) calculável cruzando dados financeiros com dados do CRM
- [ ] Documentação completa do modelo de dados financeiro no BI central
- [ ] Controles de segurança: dados financeiros no BI central com mesmo nível de acesso que no Totvs direto
- [ ] Processo de reconciliação automático: alertas quando dados no BI divergem do Totvs além da tolerância
- [ ] Alinhado com arquitetura definida na Story 9.2 (Epic 09)
- [ ] Validado pelo CFO (Claudio Kaizer) e pelo time de TI (Rafael Henrique) antes do go-live

## Riscos

| Risco | Mitigação |
|-------|-----------|
| Epic 09 (Story 9.2) atrasado — sem BI central para integrar | Iniciar modelagem e pipeline mesmo sem BI central pronto, integrar quando disponível |
| Dados financeiros inconsistentes entre Totvs e BI central | Reconciliação automática com alertas (T3.7.6) — CFO (Claudio Kaizer) deve monitorar nos primeiros meses |
| Segurança de dados financeiros no BI compartilhado | Controles de acesso rigorosos, dados financeiros isolados no BI mesmo que outros dados sejam compartilhados |
| Análises cruzadas (CAC, Margem) com lógica de negócio complexa | Definir lógica com CFO (Claudio Kaizer) e Diretor Comercial antes de implementar — não inventar regras |
