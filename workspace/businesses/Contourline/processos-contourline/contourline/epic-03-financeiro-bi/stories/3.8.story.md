## Descrição

O QA Gate do Epic 03 é o mais crítico de todos os epics — dados financeiros têm implicações legais, fiscais e de negócio que não permitem erros. Um dashboard financeiro com números errados, uma régua de cobrança enviando cobranças indevidas, ou dados de folha vazando para pessoas não autorizadas são cenários com consequências sérias.

Esta story executa validação rigorosa de todos os componentes do Epic 03 antes do go-live: precisão dos dados, segurança, LGPD, e funcionalidade de cada sistema implementado. O QA Gate também inclui validação financeira específica: todos os relatórios e cálculos são conferidos manualmente pelo time Contábil e Fiscal antes da aprovação final.

A story 3.8 só pode iniciar após Stories 3.2, 3.3, 3.4, 3.5, 3.6 e 3.7 estarem concluídas.

## Acceptance Criteria

- [ ] Dados do dashboard financeiro (Story 3.2) conferidos manualmente contra relatórios do Totvs — divergência máxima aceitável: 0.1%
- [ ] Relatórios automáticos (Story 3.3) validados por time Contábil (Thais Fernanda, Ricardo Nascimento) e Fiscal (Isabella Augusta, Janice Cristina) em pelo menos 1 ciclo real de fechamento
- [ ] Régua de cobrança (Story 3.4) testada sem enviar para clientes reais — validar com dados de teste que fluxo completo funciona
- [ ] Sistema de compras (Story 3.5) testado com compras simuladas — validar aprovações, comparativo e integração com Totvs
- [ ] Análise de folha (Story 3.6) com dados anonimizados validada pelo CFO (Claudio Kaizer) — verificar que dados individuais não são expostos
- [ ] Integração Totvs → BI central (Story 3.7) reconciliada — 100% dos dados testados correspondem entre as fontes
- [ ] Testes de acesso não autorizado: nenhum dado financeiro acessível sem credenciais válidas e permissões corretas
- [ ] LGPD: dados de clientes (cobrança) e colaboradores (folha) com controles adequados documentados
- [ ] Performance: dashboard carrega em < 5 segundos para consultas padrão
- [ ] Relatório de QA documentado com validação do CFO (Claudio Kaizer) e recomendação GO/NO-GO
