## Descrição

O CEO quer visibilidade financeira em tempo real — hoje o financeiro opera com relatórios mensais manuais e o CEO não tem acesso rápido a dados como posição de caixa, contas a pagar/receber, margem atual e desvio de budget.

Esta story cria o dashboard financeiro central alimentado pelo Totvs, dando ao CFO (Claudio Kaizer), ao CEO (Egio) e à liderança financeira visibilidade em tempo real (ou near-real-time, dependendo das limitações da API mapeadas na Story 3.1) dos indicadores financeiros críticos.

O dashboard é a peça central do Epic 03 — nele converge a inteligência financeira: dados do Totvs (Story 3.1), alertas de desvio de budget, e integração com o BI central da empresa (Story 3.7).

**Definição clara:** IA é uma camada SOBRE o Totvs. O Totvs permanece como sistema de registro. O dashboard é leitura, não escrita no Totvs.

---

## Acceptance Criteria

- [ ] Dashboard financeiro operacional com dados do Totvs (via API ou exportação conforme Story 3.1)
- [ ] KPIs principais visíveis: posição de caixa atual, saldo de contas a pagar (7/15/30 dias), saldo de contas a receber, DRE simplificado do mês corrente, margem bruta e líquida
- [ ] Comparativo: mês atual vs. mês anterior, mês atual vs. orçamento (se orçamento disponível)
- [ ] Alertas automáticos para CFO (Claudio Kaizer) / CEO quando desvio de qualquer KPI principal ultrapassa 15% do budget
- [ ] Visualização de fluxo de caixa projetado para os próximos 30, 60 e 90 dias
- [ ] Acesso segmentado por perfil: CFO vê tudo, outros financeiros veem seu escopo, CEO tem visão executiva
- [ ] Atualização de dados: no máximo a cada 1 hora (real-time se API permitir, near-real-time via exportação se não)
- [ ] Dashboard validado pelo CFO Claudio Kaizer antes do go-live
- [ ] Todos os cálculos auditáveis: cada número no dashboard tem drill-down para os dados de origem no Totvs

## Riscos

| Risco | Mitigação |
|-------|-----------|
| Dados do Totvs inconsistentes entre módulos | Reconciliação automática com alertas de divergência (implementar em T3.2.4) |
| CFO não valida os números (perde confiança) | Workshop inicial (T3.2.1) para definir o que o CFO precisa, validação rigorosa antes do go-live (T3.2.10) |
| Segurança de dados financeiros sensíveis | Criptografia em trânsito e em repouso, controle de acesso rigoroso, auditoria de acesso |
| API do Totvs muda e quebra o dashboard | Documentar versão e endpoints usados, implementar alertas de falha de coleta de dados |
