## Descrição

Após as stories técnicas do Epic 02 estarem implementadas (Stories 2.2 a 2.6), é necessário um QA Gate completo que valide o hub de marketing ponta a ponta. O sistema envolve múltiplas integrações (Hub → Eleven Labs → ferramentas de vídeo → plataformas de ads → plataformas de social media → CRM), e cada ponto de integração é um potencial ponto de falha.

O QA Gate também valida aspectos de qualidade de conteúdo: o conteúdo gerado por IA passa pelo fluxo de aprovação antes de publicação (requisito não-negociável do CEO), os dados de performance estão corretos no dashboard, e o sistema respeita as políticas de uso das APIs das plataformas.

A story 2.8 só pode iniciar após Stories 2.2, 2.3, 2.4, 2.5 e 2.6 estarem concluídas.

---

## Acceptance Criteria

- [ ] Fluxo completo testado end-to-end: briefing no hub → geração de conteúdo com IA → aprovação → agendamento → publicação no canal
- [ ] Fluxo de vídeo testado: briefing → roteiro por IA → narração Eleven Labs → edição → aprovação no hub → publicação
- [ ] Fluxo de lead testado: anúncio com UTM → chegada no RD Station com dados corretos → aparece no dashboard de atribuição
- [ ] Nenhum conteúdo publicado sem aprovação (teste de bypass de aprovação deve falhar)
- [ ] Dashboard de performance mostrando dados corretos (comparar com plataformas originais — tolerância de 5%)
- [ ] Performance: hub responde em < 3 segundos em todas as operações principais
- [ ] Integrações testadas para resiliência: o que acontece se Eleven Labs está fora? Se Instagram API retornar erro?
- [ ] Dados pessoais tratados conforme LGPD (dados de leads, dados de audiência)
- [ ] Acessos por perfil funcionando: criador vs. aprovador vs. analytics vs. admin
- [ ] Relatório de QA documentado com GO/NO-GO para go-live

## Riscos

| Risco | Mitigação |
|-------|-----------|
| Testes de social media publicam em canais reais por acidente | Usar contas de teste OBRIGATORIAMENTE — never use contas de produção |
| Falhas de APIs externas impossibilitam testes | Documentar comportamento esperado e testar com mocks quando API real não disponível |
| Issues críticos no fluxo de aprovação de conteúdo | Este é o requisito NON-NEGOTIABLE do CEO — qualquer falha aqui bloqueia o go-live |

---

## Notas para o Agente

- O requisito de aprovação humana antes de publicação é o mais crítico — testar exaustivamente todos os cenários de bypass
- Usar APENAS contas de teste para testes de social media — nunca publicar em canais reais durante QA
- Testar resiliência a falhas externas — o hub deve degradar gracefully quando APIs externas estão fora
- O relatório final deve ser compreensível para Thalison (Head de Growth Marketing), Helaine Hohendorff (Diretora de Marketing) e CEO
