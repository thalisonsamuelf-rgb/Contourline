## Descrição

A Contourline tem "muita verba de anúncio e pouca automação" — esta é uma das dores críticas mapeadas. O time de Growth (Supervisor: Thalison, Analista de Performance: Arthur) opera tráfego pago (Meta Ads, Google Ads, possivelmente LinkedIn Ads para o B2B médico) mas os leads gerados chegam ao CRM (RD Station, gerido por Ramon — Analista de CRM) sem contexto de qual campanha os gerou, qual anúncio converteu, qual segmento respondeu melhor.

O resultado: otimização de mídia paga baseada em intuição, não em dados de funil completos. O ROI de mídia paga está abaixo do potencial declarado na meta (+30%).

Esta story fecha o loop entre tráfego pago e o funil de vendas: lead gerado por anúncio X → chega ao RD Station com UTMs completos e atribuição de campanha → SDR IA qualifica → é passado para IS → fecha → o ROI real daquele anúncio é calculado. Esse dado retroalimenta o Growth para otimizar o investimento em mídia.

---

## Acceptance Criteria

- [ ] Todos os leads de tráfego pago chegam ao RD Station com UTM parameters completos (source, medium, campaign, content, term)
- [ ] Atribuição de campanha preservada do clique até o fechamento da venda (first-touch e last-touch)
- [ ] Dashboard de atribuição: qual campanha gerou quais leads, quais leads foram qualificados, quais fecharam
- [ ] ROI por campanha calculado automaticamente (custo da campanha / receita gerada pelos leads da campanha)
- [ ] Integração com pelo menos 2 plataformas de ads (Meta Ads + Google Ads como mínimo)
- [ ] Alertas para o Growth quando CPL de uma campanha ultrapassa threshold definido
- [ ] Dados de segmento do lead enriquecidos com dados da campanha (qual anúncio atraiu qual perfil)
- [ ] Integração com RD Station configurada de forma que os dados de campanha não se percam no handoff SDR → IS → FS
- [ ] Relatório semanal automático de performance de mídia paga por campanha (enviado para Growth, Thalison — Head de Growth Marketing — e Analista de Performance — Arthur)

## Riscos

| Risco | Mitigação |
|-------|-----------|
| RD Station com limitações na integração com Meta/Google | Verificar integrações disponíveis no plano atual, usar webhooks como fallback |
| UTMs sendo perdidos no funil (landing pages sem preservação) | Auditoria técnica das landing pages em T2.4.1, corrigir antes de ativar tracking |
| Dados históricos de campanhas sem UTMs impossibilitam análise retroativa | Declarar baseline como "a partir de X data" — não tentar reconciliar histórico sem UTMs |
| Custo das campanhas não centralizado (dados em cada plataforma separada) | Implementar importação manual semanal como MVP, automática como evolução |

---

## Notas para o Agente

- O tracking de UTMs é o MVP — ROI calculado vem depois, quando os dados estiverem limpos
- A convenção de UTMs (T2.4.2) precisa ser adotada pelo time de Growth — treinamento é obrigatório
- Integrar ao dashboard unificado de marketing (Story 2.5) — não criar dashboard separado
- Dados de campanha são insumo para o SDR IA (Story 1.3) — lead de campanha de demo tem perfil diferente de lead orgânico
