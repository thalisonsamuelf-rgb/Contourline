# Story 1.2 — Configuração e Integração RD Station como Hub Central

> **Epic:** 01 — Comercial & SDR
> **Status:** Draft
> **Agente Responsável:** @dev
> **Estimativa:** M (Medium)
> **Prioridade:** P1
> **Data de Criação:** 2026-03-27

---

## Descrição

O RD Station é o CRM existente da Contourline e permanece como sistema central (fora do escopo substituí-lo). 

Esta story configura o RD Station como hub central e fonte única de verdade para o funil comercial, garantindo que dados limpos e estruturados estejam disponíveis para a automação SDR (Story 1.3), os dashboards (Story 1.5) e o forecasting (Story 1.6). A auditoria técnica da API do RD Station também é realizada aqui — resultado crítico para avaliar o que é possível automatizar nas stories seguintes.

---

## Acceptance Criteria

- [ ] Estágios do funil no RD Station alinhados com o processo real mapeado na Story 1.1
- [ ] Campos obrigatórios por estágio configurados (dados mínimos para qualificação SDR)
- [ ] API do RD Station auditada: endpoints disponíveis, rate limits, autenticação, webhooks documentados
- [ ] Webhook configurado para notificar sistema externo quando lead muda de estágio
- [ ] Integração bidirecional com Marketing validada (leads gerados pelo Marketing aparecem automaticamente)
- [ ] Regras de deduplicação de leads configuradas no CRM
- [ ] Documento técnico de integração publicado (endpoints, payloads, limitações encontradas)
- [ ] Plano B documentado caso API tenha limitações críticas (ex: uso de webhooks + exportação)
- [ ] Testes de integração executados e aprovados (>= 95% de sucesso nos webhooks de teste)

---

## Tasks Técnicas

- [ ] **T1.2.1** — Auditar API do RD Station: mapear todos os endpoints disponíveis, autenticação (OAuth/API Key), rate limits, webhooks suportados
- [ ] **T1.2.2** — Revisar estágios atuais do funil no RD Station vs. mapeamento da Story 1.1, listar divergências
- [ ] **T1.2.3** — Reconfigurar etapas do funil no RD Station para refletir o processo real (SDR → IS → FS → Fechamento)
- [ ] **T1.2.4** — Definir e configurar campos obrigatórios por estágio (CNPJ, porte da empresa, produto interesse, região, origem do lead)
- [ ] **T1.2.5** — Configurar webhooks de mudança de estágio (para consumo na Story 1.3 e 1.4)
- [ ] **T1.2.6** — Criar script de limpeza de leads duplicados e leads sem dados mínimos
- [ ] **T1.2.7** — Testar integração com fontes de Marketing (formulários, landing pages, tráfego pago)
- [ ] **T1.2.8** — Documentar limitações encontradas na API e registrar plano B (webhooks, exportação periódica, etc.)
- [ ] **T1.2.9** — Criar documentação técnica de integração (endpoints usados, payloads de exemplo, autenticação)
- [ ] **T1.2.10** — Executar suite de testes de integração e registrar resultados

---

## Riscos

| Risco | Mitigação |
|-------|-----------|
| API do RD Station com limitações de integração | Auditoria técnica é a Task 1.2.1 — documentar limitações e acionar plano B (webhooks) antes de continuar |
| Reconfiguração do CRM quebrar processos atuais | Fazer em ambiente de homologação primeiro, comunicar time comercial antes de ir para produção |
| Dados existentes no CRM inconsistentes | Script de limpeza (T1.2.6) obrigatório antes de ativar automações |
