## Descrição

O time jurídico/compliance da Contourline já usa Chat PDF para consultas pontuais de normativas Anvisa — um uso pioneiro de IA para uma empresa do setor. O roadmap estratégico prevê o upgrade desse uso pontual para um sistema de monitoramento contínuo: um agente especializado em normativas Anvisa que monitora o portal oficial da Anvisa, detecta mudanças regulatórias relevantes, alerta o time e responde consultas em tempo real.

Esta story implementa a expansão do Chat PDF para um sistema mais robusto: (1) monitoramento automático do portal Anvisa via N8N com scraping periódico, (2) agente de consulta de normativas alimentado pela base de documentos regulatórios da Contourline, (3) alertas automáticos quando normas relevantes são publicadas ou alteradas.

O princípio fundamental é HUMAN-IN-THE-LOOP: o sistema suporta, informa e alerta, mas nenhuma decisão regulatória é tomada sem validação humana do time jurídico/compliance.

Stack: N8N (scraping e monitoramento) + Supabase (base de dados regulatórios) + OpenAI/Claude API (agente de consulta) + Metabase (dashboard).

## Acceptance Criteria

- [ ] Sistema de scraping do portal Anvisa (Sanvisa.gov.br) rodando diariamente e detectando publicações novas
- [ ] Base de documentos regulatórios da Contourline indexada e pesquisável por IA (mínimo: todas as RDCs e Resoluções relevantes ao portfólio)
- [ ] Agente de consulta responde perguntas sobre normativas Anvisa em < 2 minutos com citação da fonte
- [ ] Alertas automáticos enviados ao time quando norma relevante é publicada ou alterada
- [ ] Classificação automática de relevância: CRÍTICO (impacto direto no portfólio) / RELEVANTE / INFORMATIVO
- [ ] HUMAN-IN-THE-LOOP: toda resposta do agente inclui aviso de que deve ser validada pelo time jurídico
- [ ] Log de todas as consultas e alertas no Supabase (auditabilidade)
- [ ] Tempo de resposta a consulta Anvisa: de horas/dias para < 30 minutos

## Tasks Técnicas

**T8.2.1 — Definir escopo do monitoramento de normativas**
- Com time jurídico/compliance: quais tipos de publicações monitorar?
  - RDC (Resolução da Diretoria Colegiada)
  - IN (Instrução Normativa)
  - Notas Técnicas
  - Consultas Públicas
  - Comunicados de Alerta
- Filtros por categoria: dispositivos médicos, equipamentos de diagnóstico, estética
- Validar com @architect o modelo de dados para normativas

**T8.2.2 — Implementar scraping do portal Anvisa no N8N**
- Workflow N8N: cron diário (6h e 18h) acessa portal Anvisa
- Scraping de: legislação.anvisa.gov.br, consultas-publicas, alerts
- Parser de publicações novas (comparar com últimas 24h)
- Extração de: título, número, data, categoria, link, texto completo (quando disponível)
- Error handling: se scraping falhar, alerta para TI em 1h

**T8.2.3 — Criar schema de normativas no Supabase**
- Tabela `compliance_normativas` com: id, tipo (RDC/IN/etc), numero, titulo, data_publicacao, data_vigencia, categoria, texto_completo, relevancia_contourline, status_monitoramento, criado_em
- Tabela `compliance_alertas` com: normativa_id, classificacao (CRITICO/RELEVANTE/INFORMATIVO), enviado_para, data_envio, status_leitura
- Tabela `compliance_consultas` com: pergunta, resposta_ia, fontes_citadas, validado_por_humano, validado_em, usuario_consulta

**T8.2.4 — Implementar classificação de relevância com IA**
- Após cada publicação nova: enviar título + ementa para OpenAI/Claude API
- Prompt especializado: "Esta norma é relevante para uma empresa que distribui equipamentos médicos estéticos certificados pela Anvisa? Classifique como CRÍTICO (impacto direto no portfólio), RELEVANTE (pode impactar processos) ou INFORMATIVO (contexto geral). Justifique em 2 linhas."
- Gravar classificação no Supabase
- Threshold: se CRÍTICO → alerta imediato; RELEVANTE → relatório diário; INFORMATIVO → log apenas

**T8.2.5 — Implementar agente de consulta de normativas**
- Indexar base de documentos regulatórios no Supabase (vector embeddings via pgvector ou solução equivalente)
- Documentos a indexar: normativas da `compliance_normativas` + documentos de certificações da story 8.1
- Endpoint de consulta via N8N: recebe pergunta → busca documentos relevantes → monta contexto → chama LLM → retorna resposta com fontes
- Interface: inicialmente via e-mail (pergunta ao sistema → resposta em < 30 min)
- Aviso obrigatório em toda resposta: "Esta resposta é gerada por IA e deve ser validada pelo time jurídico antes de qualquer ação regulatória."

**T8.2.6 — Configurar alertas de mudanças regulatórias**
- Alert CRITICO: disparo imediato por e-mail para: Julia Maria (Analista Juridico) + Valdirene Gomes (Compliance) + CEO
- Alert RELEVANTE: relatório diário consolidado às 8h para time jurídico
- Alert INFORMATIVO: relatório semanal para assinantes interessados
- Template de alerta com: o que mudou, por que é relevante para Contourline, ação recomendada, link para texto completo

**T8.2.7 — Criar interface de consulta por e-mail**
- E-mail para endereço dedicado (ex: normas@contourline.com.br)
- N8N lê caixa de entrada periodicamente (ou via webhook Gmail)
- Extrai pergunta, processa via agente, responde ao remetente em < 30 min
- Log da consulta no Supabase

**T8.2.8 — Criar dashboard de normativas no Metabase**
- Publicações monitoradas por mês
- Distribuição por classificação (CRÍTICO/RELEVANTE/INFORMATIVO)
- Alertas enviados vs lidos
- Consultas realizadas (volume e tempo de resposta)
- Timeline de publicações relevantes (12 meses)

**T8.2.9 — Validar com time jurídico/compliance**
- Revisão de 10 classificações automáticas de relevância: corretas?
- Teste de 5 consultas reais do time ao agente: respostas adequadas?
- Ajuste de prompts e thresholds com base no feedback
- Treinamento do time no uso do sistema

## Riscos

| Risco | Probabilidade | Impacto | Mitigação |
|-------|--------------|---------|-----------|
| Portal Anvisa bloquear scraping automático | Média | Alto | Usar rate limiting (1 req/min); robots.txt; como fallback: RSS quando disponível |
| IA classificar normativa CRÍTICA como INFORMATIVA | Baixa | Crítico | Checagem manual semanal; alertas de volume (se semana sem CRÍTICO, verificar) |
| Time jurídico usar resposta da IA sem validar | Média | Crítico | Aviso obrigatório em bold em toda resposta; treinamento explícito |
| Custo de API LLM elevado com volume de normativas | Baixa | Médio | Processar apenas normativas novas; limite diário de tokens |
| pgvector não disponível no Supabase tier utilizado | Baixa | Médio | Usar Supabase tier que suporta extensões; alternativa: Pinecone gratuito |
