## Descrição

Esta story é o QA Gate de encerramento do Epic 08. Dado que o compliance Anvisa é o MOAT competitivo da Contourline, este QA Gate tem um nível de criticidade mais alto que o padrão: qualquer falha em mecanismos de alerta regulatório ou em dados de certificações deve ser tratada como FAIL imediato, não como CONCERNS.

O QA Gate valida que: (1) o inventário de certificações está completo e correto, (2) o sistema de monitoramento de normativas está detectando publicações reais, (3) os alertas de vencimento de contratos funcionam, (4) o dashboard de compliance exibe dados confiáveis, e (5) o fluxo de homologação está rastreando produtos corretamente.

Também valida o princípio HUMAN-IN-THE-LOOP: o sistema suporta e informa, mas não toma decisões regulatórias sozinho.

## Checklist de Validação Regulatória

### 7 Quality Checks do QA Gate

**QC1 — Completude Regulatória (Crítico)**
- [ ] 100% dos produtos comercializados têm registro Anvisa mapeado (story 8.1)
- [ ] AFE da empresa mapeada e com data de vencimento cadastrada
- [ ] Nenhum campo de data de vencimento em branco para itens CRÍTICOS
- [ ] Inventario aprovado formalmente por Julia Maria (Analista Juridico)

**QC2 — Monitoramento de Normativas (Crítico)**
- [ ] Scraping do portal Anvisa executado com sucesso nas últimas 48h (verificar logs N8N)
- [ ] Verificar se publicação recente real (últimos 30 dias) foi detectada e classificada
- [ ] Classificação de relevância validada: ao menos 5 publicações revisadas manualmente pelo time jurídico
- [ ] Alertas CRÍTICOS sendo entregues em < 1h da detecção

**QC3 — Gestão de Contratos (Alta)**
- [ ] Contratos críticos com vencimento nos próximos 90 dias identificados e com alertas ativos
- [ ] Testar alerta de 30 dias (simular data de vencimento)
- [ ] Verificar que contratos com dados incompletos geram aviso no dashboard
- [ ] Zero contratos vencidos sem registro no sistema

**QC4 — Dashboard de Compliance (Alta)**
- [ ] Score de risco regulatório calculado corretamente (validar manualmente com 10 items)
- [ ] Semáforo global refletindo o estado real do portfólio
- [ ] Todos os 5 painéis carregando em < 5 segundos
- [ ] Dados do dashboard conferem com queries diretas no Supabase (spot check de 20 registros)

**QC5 — Human-in-the-Loop (Crítico)**
- [ ] Toda resposta do agente de consulta Anvisa inclui aviso de validação humana obrigatória
- [ ] Nenhum fluxo automático toma decisão regulatória sem aprovação manual registrada
- [ ] Pipeline de homologação exige aprovação humana em cada transição de fase
- [ ] Log de todas as decisões automáticas auditável no Supabase

**QC6 — Segurança e Privacidade (Crítico)**
- [ ] RLS no Supabase: usuário sem permissão não consegue acessar dados de contratos ou certificações (teste com usuário de teste)
- [ ] Metabase com autenticação obrigatória (não público)
- [ ] Logs de acesso a dados sensíveis registrados
- [ ] Credenciais de API (LLM, scraping) não expostas em código ou logs

**QC7 — Operabilidade (Alta)**
- [ ] Time de TI (3 pessoas) consegue operar todo o sistema sem dependência externa
- [ ] Documentação técnica cobrindo: como adicionar nova certificação, como adicionar novo contrato, como recuperar de falha de scraping, como fazer backup dos dados
- [ ] Todos os flows N8N com tratamento de erro e alerta de falha para TI
- [ ] Rollback documentado: se sistema falhar, processo manual de contingência definido

- Verificar documentação técnica: está completa e testada?

**T8.6.9 — Documentar e emitir veredicto**
- Consolidar resultado de cada QC
- Classificar por: PASS / CONCERNS / FAIL
- Emitir veredicto global com justificativa

## Política de Veredicto (Epic 08 — Compliance)

Dado o nível de criticidade regulatória, esta política é mais rígida que o padrão:

| Resultado | Condição | Ação |
|-----------|----------|------|
| PASS | Todos os QCs PASS ou CONCERNS mínimos | Epic 08 DONE; iniciar Epic 10 |
| CONCERNS | QCs não-críticos com concerns; todos os críticos PASS | DONE com ressalvas; monitorar |
| FAIL | Qualquer QC marcado como CRÍTICO com FAIL | BLOQUEADO; retornar para @dev; re-executar QA Gate completo |

**QCs CRÍTICOS (FAIL = bloqueio obrigatório):** QC1, QC2, QC5, QC6
