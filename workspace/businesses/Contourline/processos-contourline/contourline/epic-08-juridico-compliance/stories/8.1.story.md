## Descrição

A certificação Anvisa é o MOAT competitivo da Contourline — é o que protege R$115-120M de faturamento anual e impede que concorrentes entrem no mesmo mercado. Perder uma certificação pode paralisar linhas inteiras de produto. Apesar disso, não há um inventário centralizado e atualizado de todas as certificações, registros, licenças e obrigações regulatórias da empresa.

Esta story cria o inventário completo e auditado de todas as obrigações regulatórias Anvisa da Contourline: registros de produto, AFE (Autorização de Funcionamento de Empresa), licenças específicas por categoria de equipamento, prazos de renovação, documentação associada e responsável por cada item. Este artefato é a base para o sistema de monitoramento (story 8.2) e o dashboard de compliance (story 8.4).

O time jurídico/compliance já usa Chat PDF para consultas pontuais de normativas Anvisa — este mapeamento deve ser alimentado com o conhecimento já existente no time.

- [ ] Licenças específicas por categoria de equipamento listadas (equipamentos de diagnóstico, terapêuticos, estéticos)

- [ ] Documento validado por Julia Maria (Analista Juridico) e Valdirene Gomes (Compliance) e aprovado pela lideranca
- [ ] Dados estruturados prontos para importação no Supabase (story 8.2)

## Tipos de Obrigações a Mapear

| Tipo | Descrição | Frequência |
|------|-----------|------------|
| Registro de Produto | Cada equipamento comercializado deve ter registro Anvisa | Renovação periódica (5-10 anos) |
| AFE | Autorização de Funcionamento de Empresa | Anual ou conforme mudanças |
| Licença de Importação | Para produtos importados | Por importação ou anual |
| Tecnovigilância | Reporte de incidentes adversos com equipamentos | Contínuo (quando ocorre) |
| Auditorias Anvisa | Inspeções regulatórias | Conforme cronograma Anvisa |
| IQCM | Implementação de boas práticas de fabricação | Monitoramento contínuo |
| Rotulagem | Conformidade de rótulos e manuais com normas | A cada atualização de produto |

## Tasks Técnicas

**T8.1.1 — Preparar roteiro de entrevistas com time jurídico/compliance**
- Perguntas sobre: lista de produtos registrados, processos de renovação, ferramentas usadas, histórico de infrações (se houver), obrigações periódicas conhecidas
- Incluir perguntas sobre uso atual do Chat PDF e quais normativas são consultadas com mais frequência

**T8.1.2 — Conduzir entrevistas com Julia Maria (Analista Juridico) e Valdirene Gomes (Compliance)**
- Mínimo 2 sessões de 1h cada
- Gravação ou notas estruturadas com aprovação dos entrevistados
- Levantar: onde estão armazenados os documentos hoje (pasta local? e-mail? GED?)

**T8.1.3 — Consultar portal Anvisa para verificação dos registros**
- Acessar DATAVISA (portal público Anvisa) para verificar status de registros
- Verificar: CMED para equipamentos com questões de precificação
- Documentar discrepâncias entre o que o time sabe e o que consta no portal

**T8.1.4 — Consolidar inventário em planilha estruturada**
- Campos obrigatórios: id, tipo, numero_registro, produto, data_emissao, data_vencimento, status, risco, responsavel, documentos_urls, notas
- Um registro por linha
- Validação: todos os campos obrigatórios preenchidos

**T8.1.5 — Mapear calendário de vencimentos (24 meses)**
- Ordenar por data de vencimento
- Classificar: CRÍTICO (<6 meses), ATENÇÃO (6-12 meses), OK (>12 meses)
- Identificar clusters de vencimentos (meses com muitas renovações simultâneas)

**T8.1.6 — Mapear obrigações periódicas e processos de tecnovigilância**
- Listar todos os relatórios periódicos obrigatórios com frequência e prazo
- Documentar processo atual de reporte de tecnovigilância (quem faz, como, onde registra)
- Identificar se há sistema de rastreio de incidentes adversos

**T8.1.7 — Identificar gaps e riscos regulatórios**
- Items com vencimento em risco não detectados
- Obrigações que podem estar sendo descumpridas
- Documentação de produtos sem evidência de registro localizada

**T8.1.8 — Validar inventário com time jurídico/compliance**
- Revisão conjunta da planilha
- Aprovação formal (assinatura ou e-mail de confirmação)
- Ajustes de dados

**T8.1.9 — Preparar dados para importação no Supabase**
- Converter planilha para formato JSON/CSV compatível com schema definido na story 8.2
- Documentar estrutura dos dados para @dev
