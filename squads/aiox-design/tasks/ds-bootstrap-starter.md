# Bootstrap Design Starter

> Task ID: ds-bootstrap-starter
> Agent: Design Chief (Orchestrator)
> Version: 1.0.0
> **Execution Type:** `Automated`
> **Human Checkpoint:** Verify the starter runs with `npm run doctor` before delivery
> **Dependencies:** depends_on: `[]` · enables: `[ds-build-component]` · workflow: `standalone`

## Description

Bootstrap the standalone `aiox-design-starter` repo for a new client or environment. Clones the repo, runs setup, optionally configures a variant, and validates everything works. The starter is a self-contained Next.js 16 brandbook/design-system with 60+ components, Tailwind CSS v4, and white-label variant support.

## Output Schema
- **produces:** Functional local instance of `aiox-design-starter`
- **format:** Running Next.js app with all checks green
- **consumed_by:** Client/user for brandbook customization

## Prerequisites

- Node.js >= 18.0.0
- npm (included with Node.js)
- Git
- GitHub access to `oalanicolas/aiox-design-starter` (private repo)

## Workflow

### Step 1: Clone

```bash
git clone https://github.com/oalanicolas/aiox-design-starter.git
cd aiox-design-starter
```

### Step 2: Setup

```bash
npm run setup
```

This runs `scripts/bootstrap.cjs` which:
1. Validates Node >= 18
2. Validates npm is available
3. Runs `npm install`
4. Runs token generation (`npm run tokens:sync:write`)
5. Verifies generated files exist

### Step 3: Validate

```bash
npm run doctor
```

Expected: 17/17 checks green, "Projeto saudavel!"

### Step 4: Run

```bash
npm run dev
```

Opens http://localhost:3000 with the brandbook.

### Step 5 (Optional): Create Variant

To create a white-label variant for a specific client:

```bash
npm run scaffold:variant -- --slug=client-name --name="Client Name" --force
STARTER_VARIANT=client-name npm run dev
```

### Step 6 (Optional): Connect to Workspace

To import data from a business in the aiox-stage monorepo:

```bash
STARTER_SOURCE=workspace BUSINESS_SLUG=aiox WORKSPACE_ROOT=/path/to/aiox-stage npm run dev
```

## Available Commands

| Command | Description |
|---------|-------------|
| `npm run setup` | Full bootstrap (install + tokens) |
| `npm run dev` | Development server |
| `npm run build` | Production build |
| `npm run doctor` | Health check (17 checks) |
| `npm run typecheck` | TypeScript validation |
| `npm run lint` | ESLint check |
| `npm run scaffold:variant` | Create new white-label variant |
| `npm run tokens:sync:write` | Regenerate tokens manually |

## Customization Points

| What to change | File |
|----------------|------|
| Brand (name, logo, SEO) | `starter/site.config.yaml` |
| Theme (colors, tokens) | `starter/design-system.config.yaml` |
| Brandbook content | `starter/content/brandbook.yaml` |
| Fonts | `starter/site.config.yaml` → `fonts.external_stylesheets` |

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `npm install` fails with peer deps | `.npmrc` already has `legacy-peer-deps=true` |
| Port 3000 in use | Next.js auto-selects next available port |
| Tokens not generated | Run `npm run tokens:sync:write` |
| Typecheck errors | Run `npm run doctor` for diagnostic |

## Architecture Notes

- **Repo:** `https://github.com/oalanicolas/aiox-design-starter`
- **Stack:** Next.js 16, Tailwind CSS v4, shadcn/ui, Radix UI
- **Packages:** `brandbook-primitives` and `brandbook-editorial` internalized in `packages/`
- **Tenant system:** Supports `starter` (local-first, zero config) and `workspace` (monorepo bridge) modes
- **Generated files:** `*.generated.css` and `*.generated.ts` are gitignored, regenerated by `npm run setup`
