## DETAILED CHECKLIST BY EMAIL

### EMAIL 1: SET THE STAGE

```yaml
email_1_checklist:
  hook:
    - [ ] Primeira linha captura atenção imediatamente
    - [ ] Não começa com saudação genérica
    - [ ] Cria curiosidade ou intrigue

  character_introduction:
    - [ ] Protagonista é claramente identificado
    - [ ] Detalhes tornam o personagem relatable
    - [ ] Audiência pode se identificar com a situação

  status_quo:
    - [ ] Situação inicial está clara
    - [ ] O "antes" é estabelecido
    - [ ] Há hints de que algo está para mudar

  tension_building:
    - [ ] Pequenos sinais de problema surgindo
    - [ ] Leitor sente que algo vai acontecer
    - [ ] Não revela o conflito completo

  open_loop:
    - [ ] Cliffhanger é específico
    - [ ] Cria urgência para ler o próximo
    - [ ] Promete valor no Email 2

  subject_line:
    - [ ] Curto (< 50 caracteres)
    - [ ] Cria curiosidade
    - [ ] Conecta com o conteúdo

  overall:
    - [ ] Leitor quer saber o que aconteceu depois
    - [ ] Parágrafos são curtos
    - [ ] Linguagem é conversacional
```

### EMAIL 2: HIGH DRAMA

```yaml
email_2_checklist:
  bridge:
    - [ ] Conecta ao Email 1
    - [ ] Recap é breve (não repete tudo)
    - [ ] Transição é natural

  conflict_intensification:
    - [ ] Problema se manifesta claramente
    - [ ] Conflito é intensificado (não resolvido)
    - [ ] Leitor entende o que deu errado

  emotional_depth:
    - [ ] Usa detalhes sensoriais
    - [ ] Inclui diálogo ou pensamentos internos
    - [ ] Leitor SENTE a dor (não só entende)

  stakes:
    - [ ] O que está em risco é claro
    - [ ] Consequências de não resolver são mostradas
    - [ ] Há senso de urgência na situação

  failed_attempts:
    - [ ] Mostra soluções convencionais tentadas
    - [ ] Explica por que falharam
    - [ ] Valida a frustração do leitor

  turning_point_tease:
    - [ ] Há hint de que algo vai mudar
    - [ ] Esperança surge (mas sem revelar)
    - [ ] Cliffhanger para Email 3

  overall:
    - [ ] Leitor está emocionalmente investido
    - [ ] Drama é autêntico, não artificial
    - [ ] Urgência para ler Email 3
```

### EMAIL 3: EPIPHANY

```yaml
email_3_checklist:
  bridge:
    - [ ] Recap rápido do drama
    - [ ] Conecta ao momento de desespero

  epiphany_moment:
    - [ ] Momento é específico e vívido
    - [ ] Detalhes de onde/quando/como
    - [ ] O "clique" é claro

  insight:
    - [ ] Descoberta é explicada claramente
    - [ ] Leitor entende o que foi descoberto
    - [ ] Insight faz sentido lógico

  why_it_works:
    - [ ] Princípio por trás é explicado
    - [ ] Desafia sabedoria convencional
    - [ ] Leitor pode ver por que isso muda tudo

  early_results:
    - [ ] Primeiros resultados são mostrados
    - [ ] Específico (não vago)
    - [ ] Crível (não exagerado)

  transformation_tease:
    - [ ] Hint de mais benefícios por vir
    - [ ] "Mas isso era só o começo..."
    - [ ] Cliffhanger para Email 4

  overall:
    - [ ] Leitor tem momento "aha" próprio
    - [ ] Quer aplicar o insight
    - [ ] Ansioso pelo Email 4
```

### EMAIL 4: HIDDEN BENEFITS

```yaml
email_4_checklist:
  bridge:
    - [ ] Recap da epifania
    - [ ] Transição para resultados

  transformation:
    - [ ] Resultados específicos mostrados
    - [ ] Números quando possível
    - [ ] Mudanças tangíveis descritas

  hidden_benefits:
    - [ ] Benefícios inesperados revelados
    - [ ] Surpresas positivas
    - [ ] "O que eu não esperava..."

  social_proof:
    - [ ] Outros resultados (se disponível)
    - [ ] Depoimentos (se possível)
    - [ ] Validação do método

  objection_handling:
    - [ ] Objeções comuns endereçadas
    - [ ] Através da história (não sermão)
    - [ ] "Eu também pensei que..."

  product_tease:
    - [ ] Menção de que sistematizou
    - [ ] Hint de oferta chegando
    - [ ] Natural, não abrupto

  cliffhanger:
    - [ ] Prepara para Email 5
    - [ ] Cria expectativa para oferta
    - [ ] "Amanhã..."

  overall:
    - [ ] Leitor quer o que você tem
    - [ ] Objeções foram trabalhadas
    - [ ] Pronto para a oferta
```

### EMAIL 5: URGENCY

```yaml
email_5_checklist:
  recap:
    - [ ] Jornada resumida brevemente
    - [ ] Conecta os 4 emails anteriores
    - [ ] Leitor relembra a história

  loop_closure:
    - [ ] TODOS os loops são fechados
    - [ ] Nenhuma promessa ficou pendente
    - [ ] Satisfação emocional

  offer_presentation:
    - [ ] Oferta é apresentada claramente
    - [ ] Conecta naturalmente à história
    - [ ] Não parece venda forçada

  value_stack:
    - [ ] O que está incluído é claro
    - [ ] Benefícios de cada componente
    - [ ] Valor percebido > preço

  risk_reversal:
    - [ ] Garantia oferecida
    - [ ] Medo de comprar removido
    - [ ] Confiança aumentada

  urgency:
    - [ ] Razão GENUÍNA para agir agora
    - [ ] Deadline ou escassez REAL
    - [ ] Consequência de esperar clara

  cta:
    - [ ] Chamada para ação clara
    - [ ] Link visível e funcional
    - [ ] Leitor sabe exatamente o que fazer

  overall:
    - [ ] Oferta é conclusão natural
    - [ ] Urgência não parece manipulação
    - [ ] Leitor motivado a clicar
```

## RED FLAGS (Se qualquer um = REVISAR)

### Story Red Flags

```
❌ História é genérica (poderia ser qualquer pessoa)
❌ Conflito parece fabricado ou artificial
❌ Epifania é clichê ou óbvia demais
❌ Transformação é exagerada ou increível
❌ Protagonista não tem falhas
```

### Structure Red Flags

```
❌ Loops não são fechados no Email 5
❌ Oferta parece desconectada da história
❌ Urgência é claramente falsa
❌ Cliffhangers são genéricos ("mais amanhã")
❌ Salto direto para venda (sem construção)
```

### Engagement Red Flags

```
❌ Parágrafos longos (> 3 linhas)
❌ Linguagem corporativa ou formal
❌ Falta de detalhes específicos
❌ Sem diálogo ou pensamentos internos
❌ Hooks fracos ou genéricos
```
