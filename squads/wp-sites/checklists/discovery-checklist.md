# Discovery Completeness Checklist

**Checklist ID:** `discovery-checklist`
**Phase:** Discovery
**Agent:** wp-discovery
**Gate:** Client approval required before proceeding to Architecture

## Business Context
- [ ] Business description is clear and specific (one sentence)
- [ ] Competitors identified (at least 2)
- [ ] Differentiator stated
- [ ] Primary business goal for website defined and measurable

## Target Audience
- [ ] Ideal customer profile defined
- [ ] Primary pain point identified
- [ ] Search behavior documented (what they search for)
- [ ] Key objections listed

## Content & Pages
- [ ] Page list complete (all required pages identified)
- [ ] Content migration needs assessed
- [ ] Blog need determined (yes/no with rationale)
- [ ] Portfolio/cases need determined

## Functionality
- [ ] Form requirements documented
- [ ] E-commerce need determined (yes/no)
- [ ] Integration requirements listed
- [ ] Special features documented

## Visual Preferences
- [ ] Brand assets inventory (logo, colors, fonts -- available or needed)
- [ ] At least 1 reference site provided and analyzed
- [ ] Visual dislikes documented (what to avoid)

## Constraints
- [ ] Target launch date documented
- [ ] Hosting and domain status known
- [ ] Technical constraints documented

## Final Validation
- [ ] All sections have complete answers (no blanks)
- [ ] Objectives extracted and structured
- [ ] Brief formatted and saved to project docs
- [ ] Client has reviewed the brief
- [ ] Client has approved the brief (gate)
