## Overview

Create section-level wireframe skeletons for every page. Each skeleton defines
the section order, section purpose, and mobile stack order. These are NOT visual
mockups -- they define information flow.

## Action Items (per page)

### Step 1: Select Page Framework
- [ ] LP Product/Service: Hero -> Problem -> Solution -> Social Proof -> CTA
- [ ] Institutional: Content sections appropriate to page type
- [ ] Lead Capture: Hook -> Benefits -> Short Form

### Step 2: Define Sections
- [ ] List sections in order for the page
- [ ] Each section has: name, purpose, expected content type
- [ ] Define mobile stack order (may differ from desktop)

### Step 3: Document Wireframe
- [ ] Write wireframe skeleton in text format
- [ ] Include notes for copywriter (what copy goes in each section)
- [ ] Include notes for designer (layout hints, emphasis points)

## Acceptance Criteria

- [ ] **AC-1:** Wireframe skeleton exists for every page in sitemap
- [ ] **AC-2:** Each section has a clear purpose statement
- [ ] **AC-3:** Mobile section order is explicitly defined
- [ ] **AC-4:** Wireframes follow the assigned page framework pattern
